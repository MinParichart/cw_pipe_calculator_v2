## ภาพรวมระบบ

### วัตถุประสงค์

คำนวณขนาดท่อน้ำดีสำหรับอาคารที่พักอาศัยสูงไม่เกิน 2 ชั้น โดยใช้หลักการ:

- **Hunter's Curve** - แปลง Fixture Units (FU) เป็นอัตราการไหล (GPM)
- **Hazen-Williams Formula** - คำนวณความดันลดในท่อ (Friction Loss)
- **ตาราง 2.6** - เลือกขนาดท่อตามมาตรฐาน

### ขอบเขตการคำนวณ

- **Critical Path**: ใช้สูตร Hazen-Williams คำนวณรายละเอียด
- **Non-Critical Path**: ใช้ตาราง 2.6 เลือกขนาดท่อโดยควบคุมความเร็ว

---

## หลักการคำนวณหลัก

### 1. Hunter's Curve

แปลงค่า Fixture Units (FU) เป็นอัตราการไหล (GPM) โดย:

- **Flush Tank System**: สำหรับระบบถังน้ำ
- **Flush Valve System**: สำหรับระบบวาล์ว

### 2. Hazen-Williams Formula

คำนวณความดันลดในท่อ (Head Loss)

#### หน่วย SI (เมตร)

```
hf = (10.583 / D^4.87) × L × (Q / C)^1.85
```

- **hf** = ความดันลด (เมตรน้ำ)
- **D** = เส้นผ่านศูนย์กลางท่อ (เมตร)
- **L** = ความยาวท่อ (เมตร)
- **Q** = อัตราการไหล (m³/s)
- **C** = สัมประสิทธิ์ความหยาบ

### 3. การควบคุมความเร็ว (Velocity Control)

- **ความเร็วที่เหมาะสม**: 1.2 - 2.4 m/s
- **ความเร็วสูงสุด**: ไม่เกิน 3 m/s

## ขั้นตอนการคำนวณ Critical Path

### ภาพรวม 16 ขั้นตอน

```
จุดเริ่มต้น (Source) → กำหนด Nodes → นับ FU → แปลงเป็น Flow Rate →
คำนวณความเร็ว → คำนวณ Head Loss → รวมทั้งหมด → กำหนดขนาดปั๊ม →
สิ้นสุด (Pump Specification)
```

### รายละเอียด 16 ขั้นตอน

#### ขั้นตอนที่ 1: กำหนด Critical Path (CP)

**เป้าหมาย**: ระบุเส้นทางที่ยาวและสูงที่สุดจากปั๊มถึงสุขภัณฑ์ปลายทาง

**วิธีการ**:

- วาดแผนภาพระบบท่อแบบ Tree Diagram
- ระบุจุดเริ่มต้น (Source) - จุดจ่ายน้ำที่ออกจากปั๊ม
- ระบุจุดปลายทาง (Endpoints) - สุขภัณฑ์แต่ละตัว
- คำนวณระยะทางจาก Source ถึงแต่ละ Endpoint
- เลือกเส้นทางที่มีระยะทางรวมยาวที่สุด

#### ขั้นตอนที่ 1.1: กำหนด Nodes และแบ่งช่วงท่อ (Pipe Segments)

**เป้าหมาย**: แบ่งระบบออกเป็นส่วนๆ ที่เรียกว่า "Segments"

**วิธีการ**:

- สร้าง Node ที่จุดเปลี่ยนทิศทางหรือจุดแยก
- แบ่งท่อออกเป็น Segment ระหว่าง Nodes สองจุด
- ตั้งชื่อ Segment เช่น AB, BC, CD, DE...
- วัดความยาวของแต่ละ Segment (L)

**ตัวอย่าง**:

```
Source (A) → Segment AB → Node B → Segment BC → Node C → ... → Endpoint K
```

#### ขั้นตอนที่ 1.2: นับค่า Fixture Units (FU) ในแต่ละช่วงท่อ

**เป้าหมาย**: รวมโหลดสะสมจากปลายทางย้อนกลับมา

**วิธีการ**:

- เริ่มจากปลายทางสุด (Endpoint)
- นับ FU ของสุขภัณฑ์ในแต่ละสาย
- สะสม FU ย้อนกลับมาทาง Source
- บันทึกผลรวม FU สะสมของแต่ละ Segment

**ตัวอย่าง**:

```
Segment KL: 1 WC (3 FU) = 3 FU
Segment JK: 1 Lavatory (1 FU) + KL (3 FU) = 4 FU
Segment IJ: 1 Shower (2 FU) + JK (4 FU) = 6 FU
...
```

#### ขั้นตอนที่ 1.3: แปลงค่า FU เป็นอัตราการไหล (Q, GPM)

**เป้าหมาย**: ใช้ Hunter's Curve แปลง FU → GPM

**วิธีการ**:

- ใช้ตาราง Hunter's Curve (ตาราง 2.3)
- เลือกตารางตามประเภทระบบ (Flush Tank / Flush Valve)
- ทำ Interpolation หากค่า FU อยู่ระหว่างค่าในตาราง

**สูตร Interpolation**:

```
Q = Q1 + (FU - FU1) × (Q2 - Q1) / (FU2 - FU1)
```

**ตัวอย่าง**:

```
FU = 34 (ระหว่าง 30 และ 35)
Q = 20 + (34 - 30) × (22.5 - 20) / (35 - 30) = 22 GPM
```

#### ขั้นตอนที่ 1.4: ปรับค่าอัตราการไหล (Q, GPM) ด้วย Water Factor

**เป้าหมาย**: ปรับค่า Flow Rate ตามประเภทอาคาร

**วิธีการ**:

- ดูตาราง Water Factor (ตาราง 2.4 หรือ 2.5)
- เลือกตารางตามประเภทอาคาร (Office, Hospital, Apartment)
- คูณ GPM ด้วยเปอร์เซ็นต์ที่กำหนด

**ตัวอย่าง**:

```
อาคารพักอาศัย (Apartment):
FU = 800, Hunter GPM = 62
Water Factor = 55% → Adjusted GPM = 62 × 0.55 = 34.1 GPM
```

#### ขั้นตอนที่ 1.5: บวกเพิ่มอัตราการไหลของ Hose Bibb (HB)

**เป้าหมาย**: เพิ่ม Flow Rate ของ Hose Bibb เพราะไม่นับรวมใน FU ปกติ

**หมายเหตุ**:

- Hose Bibb มักไม่รวมในการคำนวณ FU ปกติ
- ต้องบวก Flow Rate ของ HB เข้าไปโดยตรง

**ค่าที่ใช้**:

- Hose Bibb Flow Rate = **5 GPM** (ปกติ)
- หรือตามที่กำหนดในตาราง 2.1

**ตัวอย่าง**:

```
Adjusted Flow (จากขั้นตอน 1.4) = 10 GPM
Hose Bibb Flow = 15 GPM
Total Flow = 10 + 15 = 25 GPM
```

#### ขั้นตอนที่ 1.6: แปลงหน่วยอัตราการไหล (Q) จาก GPM → LPS → m³/s

**เป้าหมาย**: แปลงหน่วยเพื่อใช้ในสูตรคำนวณหน่วย SI

**สูตรแปลงหน่วย**:

```
1 GPM = 0.06309 LPS
1 LPS = 0.001 m³/s
ดังนั้น: 1 GPM = 0.00006309 m³/s
```

**ตัวอย่าง**:

```
Flow = 15 GPM
Flow (LPS) = 15 × 0.06309 = 0.94635 LPS
Flow (m³/s) = 15 × 0.00006309 = 0.00094635 m³/s
```

#### ขั้นตอนที่ 1.7: กำหนดขนาดท่อ (D, in) เบื้องต้นจากตาราง 2.6

**เป้าหมาย**: เลือกขนาดท่อเบื้องต้นก่อนคำนวณสูตร

**วิธีการ**:

- ใช้ผลรวม FU ของแต่ละ Segment
- อ้างอิงตาราง 2.6 (Pipe Sizes)
- เลือกขนาดท่อที่เหมาะสมกับจำนวน FU

**ตัวอย่าง**:

```
Segment AB: FU = 34 → เลือกท่อ 1.5" (40 mm)
Segment BC: FU = 24 → เลือกท่อ 1.25" (32 mm)
Segment CD: FU = 12 → เลือกท่อ 1" (25 mm)
```

#### ขั้นตอนที่ 1.8: คำนวณความเร็วการไหล (v, m/s) จาก v = Q/A

**เป้าหมาย**: ตรวจสอบความเร็วน้ำในท่อ

**สูตร**:

```
v = Q / A
A = π × (D/2)²
```

- **v** = ความเร็ว (m/s)
- **Q** = อัตราการไหล (m³/s)
- **A** = พื้นที่หน้าตัดท่อ (m²)
- **D** = เส้นผ่านศูนย์กลางท่อ (m)

**เกณฑ์การตรวจสอบ**:

- ความเร็วที่เหมาะสม: **1.2 - 2.4 m/s**
- ความเร็วสูงสุด: **ไม่เกิน 3 m/s**

**ตัวอย่าง**:

```
Q = 0.00094635 m³/s
D = 1.5" = 0.0381 m
A = π × (0.0381/2)² = 0.00114 m²
v = 0.00094635 / 0.00114 = 0.83 m/s ✓
```

#### ขั้นตอนที่ 1.9: กรอกความยาวท่อจริง (L, m) และค่าสัมประสิทธิ์ความขรุขระ (C)

**เป้าหมาย**: เตรียมข้อมูลสำหรับคำนวณ Hazen-Williams

**ข้อมูลที่ต้องการ**:

1. **ความยาวท่อจริง (L)**: วัดจากแปลนหรือคำนวณจากระยะห่าง Nodes
2. **ค่า C**: เลือกตามชนิดท่อจากตาราง C-Factor

**ค่า C ที่นิยมใช้**:

- ท่อ PVC: **C = 150**
- ท่อทองแดง: **C = 130**
- ท่อเหล็ก: **C = 100**

#### ขั้นตอนที่ 1.10: คำนวณ Friction Loss Rate (h_f, m/100m)

**เป้าหมาย**: ใช้สูตร Hazen-Williams คำนวณความดันลดต่อ 100m

**สูตร**:

```
h_f = (10.583 / D^4.87) × (Q / C)^1.85 × 100
```

- **h_f** = Friction Loss Rate (m/100m)
- **D** = เส้นผ่านศูนย์กลาง (m)
- **Q** = อัตราการไหล (m³/s)
- **C** = สัมประสิทธิ์ความหยาบ

**ตัวอย่าง**:

```
D = 0.0381 m (1.5")
Q = 0.00094635 m³/s (15 GPM)
C = 110

h_f = (10.583 / 0.0381^4.87) × (0.00094635 / 110)^1.85 × 100
h_f = 3.61891 m/100m
```

#### ขั้นตอนที่ 1.11: คำนวณ Major Loss (h_l, m)

**เป้าหมาย**: คำนวณความดันลดจริงของแต่ละ Segment

**สูตร**:

```
h_l = L × (h_f / 100)
```

- **h_l** = Major Loss (m.wg)
- **L** = ความยาวท่อจริง (m)
- **h_f** = Friction Loss Rate (m/100m)

**ตัวอย่าง**:

```
L = 1.3 m
h_f = 3.61891 m/100m

h_l = 1.3 × (3.61891 / 100) = 0.04705 m.wg
```

#### ขั้นตอนที่ 1.12: คำนวณ Minor Loss จากอุปกรณ์และข้อต่อ

**เป้าหมาย**: คำนวณความดันลดจากข้อต่อ หัวเทียน วาล์ว

**วิธีการ (Simplified Method)**:

```
Minor Loss = Major Loss × 30%
```

**หมายเหตุ**:

- วิธีนี้ใช้สำหรับการคำนวณเบื้องต้น
- สำหรับการคำนวณละเอียด ควรใช้ Equivalent Length Method

#### ขั้นตอนที่ 1.13: คำนวณ Static Head (ความสูงต่างระดับ)

**เป้าหมาย**: คำนวณความดันลดเนื่องจากความสูง

**สูตร**:

```
Static Head = ความสูงจากปั๊มถึงสุขภัณฑ์ที่สูงที่สุด (เมตร)
```

**ตัวอย่าง**:

```
ปั๊มอยู่ที่ระดับน้ำตก +0.00 m
สุขภัณฑ์สูงสุดอยู่ที่ +3.50 m
Static Head = 3.50 m.wg
```

#### ขั้นตอนที่ 1.14: รวมความดันสูญเสียทั้งหมดในระบบ (Total Head Loss)

**เป้าหมาย**: สรุปความดันสูญเสียรวมทั้งหมด

**สูตร**:

```
Total Head Loss = (Major Loss + Minor Loss) + Static Head
```

**ตัวอย่าง**:

```
Major Loss (รวมทั้งหมด) = 2.128 m.wg
Minor Loss (30% ของ Major) = 0.638 m.wg
Static Head = 1.000 m.wg

Total Head Loss = (2.128 + 0.638) + 1.000 = 3.766 m.wg
```

#### ขั้นตอนที่ 1.15: กำหนดความดันใช้งานที่ปลายทาง (Residual Pressure)

**เป้าหมาย**: กำหนดความดันที่ต้องการใช้งานจริง

**ค่าที่นิยมใช้**:

- ฝักบัว (Shower): **0.5 bar** (5 m.wg)
- ก๊อกอ่างล้างหน้า: **0.5 bar** (5 m.wg)
- โถส้วม (Flush Valve): **0.7 bar** (7 m.wg)
- Hose Bibb: **0.35 bar** (3.5 m.wg)

**ตัวอย่าง**:

```
Residual Pressure (Hose Bibb) = 0.35 bar = 3.5 m.wg
```

#### ขั้นตอนที่ 1.16: สรุปแรงดันที่ปั๊มต้องทำได้ (Total Dynamic Head - TDH)

**เป้าหมาย**: กำหนดสเปคปั๊มที่เหมาะสม

**สูตร**:

```
TDH = Total Head Loss + Residual Pressure
```

**ตัวอย่าง**:

```
Total Head Loss = 3.766 m.wg (0.376 bar)
Residual Pressure = 3.5 m.wg (0.35 bar)

TDH = 3.766 + 3.5 = 7.266 m.wg
TDH (bar) = 7.266 / 10.197 = 0.713 bar
TDH (PSI) = 0.713 × 14.5 = 10.34 PSI
```

## ขั้นตอนการคำนวณ Non-Critical Path

### หลักการ

สำหรับท่อที่ไม่ใช่ Critical Path:

- **ไม่ต้องคำนวณ Hazen-Williams ละเอียด**
- **ใช้ตาราง 2.6 เลือกขนาดท่อโดยตรง**
- **ควบคุมความเร็วเป็นหลัก (Velocity Control)**

### เหตุผล

เส้นท่อมีความยาวน้อยและมีโหลดต่ำ ทำให้ความดันลด (Head Loss) มีค่าน้อยมาก จนไม่ส่งผลกระทบต่อความต้องการแรงดันของปั๊มน้ำ

### วิธีการ

1. นับ FU ของแต่ละ Segment
2. อ้างอิงตาราง 2.6
3. เลือกขนาดท่อตามจำนวน FU
4. ตรวจสอบความเร็ว (v) ไม่เกิน 3 m/s

## สูตรและค่าคงที่

### แปลงหน่วยอัตราการไหล

```
1 GPM = 0.06309 LPS = 0.00006309 m³/s
1 LPS = 15.85 GPM
1 m³/s = 15850 GPM
```

### แปลงหน่วยความดัน

```
1 bar = 10.197 m.wg (เมตรน้ำ)
1 bar = 14.5 PSI
1 PSI = 0.06895 bar
1 m.wg = 0.0981 bar
```

### แปลงหน่วยขนาดท่อ

```
1 inch = 25.4 mm
1/2" = 15 mm
3/4" = 20 mm
1" = 25 mm
1¼" = 32 mm
1½" = 40 mm
2" = 50 mm
```

### สูตรคำนวณพื้นที่หน้าตัดท่อ

```
A = π × (D/2)²
```

- **A** = พื้นที่หน้าตัด (m²)
- **D** = เส้นผ่านศูนย์กลาง (m)

### สูตรคำนวณความเร็ว

```
v = Q / A
```

- **v** = ความเร็ว (m/s)
- **Q** = อัตราการไหล (m³/s)
- **A** = พื้นที่หน้าตัด (m²)

### Velocity Head

```
hv = v² / (2 × g)
```

- **hv** = Velocity Head (m.wg)
- **v** = ความเร็ว (m/s)
- **g** = ความโน้มถ่วง = 9.81 m/s²

## ฟีเจอร์ระบบ

### A) Account & Security (2 ฟีเจอร์)

1. **Authentication**: ระบบ Login/Logout
2. **Authorization**: จำกัดสิทธิ์การเข้าถึง (เห็น/แก้ไขเฉพาะโปรเจกต์ของตัวเอง)

### B) Project & Workspace Setup (3 ฟีเจอร์)

3. **Project Management (CRUD)**: สร้าง/ดู/แก้ไข/ลบ/Duplicate Project
4. **Design Criteria Settings**: ตั้งค่าเกณฑ์คำนวณ (v, Pressure, C-Factor)
5. **Reference Data**: เปิดดูตารางอ้างอิงแบบ Read-only

### C) Versioning & Traceability (5 ฟีเจอร์)

6. **Version CRUD**: สร้าง/แก้ชื่อ/ลบ/Duplicate Version
7. **Save Version**: บันทึก Snapshot เป็นเวอร์ชัน
8. **View Version History**: ดูรายการเวอร์ชันทั้งหมด
9. **Compare Versions**: เทียบความต่าง 2 เวอร์ชัน
10. **Audit Log**: ดูประวัติการกระทำ

### D) Interactive Network Builder (4 ฟีเจอร์)

11. **Blueprint Calibration**: อัปโหลดรูปและระบุสเกล
12. **Visual Node Builder**: วาง Node (Source, Junction, Fixture, Riser)
13. **Smart Pipe Drawing**: ลากเส้นท่อเชื่อม Node พร้อมคำนวณความยาว
14. **Pathfinding & Critical Path**: ระบบหา Critical Path อัตโนมัติ

### E) Fixtures & Engineering Catalogs (4 ฟีเจอร์)

15. **Smart Fixture Mapping**: เลือกสุขภัณฑ์พร้อมดึงค่า FU อัตโนมัติ
16. **Pipe Specs Catalog**: ฐานข้อมูลท่อละเอียด (DN, ID, Pressure Class)
17. **FU Summary**: รวม FU ต่อ node และรวมทั้งระบบ
18. **Hunter's Curve Engine**: ระบบแปลง FU → Flow พร้อม Interpolation

### F) Hydraulic Sizing & Verification (2 ฟีเจอร์)

19. **Hybrid Pipe Sizing**: เลือกขนาดท่อ (Table 2.6 + Hazen-Williams)
20. **Hydraulic Verification**: ตรวจสอบ v, hf, Minor Loss, Static Head, Residual

### G) Optimization & Results (3 ฟีเจอร์)

21. **Auto-Suggest Upsizing**: ระบบแนะนำการปรับขนาดท่อ
22. **Apply Suggestion**: ปุ่มกดเลือกขนาดท่อตามที่ระบบแนะนำ
23. **Results Dashboard**: สรุปผล + สเปคปั๊ม (TDH)

## Flow การใช้งานจริง (13 ขั้นตอน)

1. Login
2. Project List → สร้าง/เลือกโปรเจกต์
3. ตั้งค่า Hydraulic Parameters + ดู Reference Data
4. Upload Blueprint + ใส่ Manual Note/Label
5. Build Network (Tree) + เลือก Critical Endpoint
6. Fixture Catalog → Assign Fixtures → ดู FU Summary
7. Water Demand Calculation (FU → Flow)
8. Pipe Sizing (เลือก DN จาก Table 2.6)
9. Hydraulic Verification (velocity/headloss/static/residual)
10. Results Dashboard (PASS/WARN/FAIL)
11. (ถ้าต้องปรับ) Auto-suggest Upsizing → Apply Suggestion
12. Save Version → View History → Compare Versions
13. ดู Audit Log
